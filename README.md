Пункт проката

Реализуйте приложение, позволяющие сотруднику пунка проката вести учёт единиц прокатного снаряжения (коньки, велосипеды и др.) и дополнительных аксессуаров (шлем, защита и др.).
В приложении должны быть реализованы следующие возможности: 
просмотр спика доступного к прокату снаряжения и аксессуаров
возможность одному клиенту взять до 3х единиц снаряжения в прокат на определённый срок (в часах или сутках)
поиск подходящего снаряжения по параметрам (вес, размер и др.)
Дополнительные условия:
1. В качестве класса, хранящего единицу снаряжения, используйте класс Equipment. (Если необходимо, расширьте иерархию спортивного снаряжения)
public class Equipment {
	private Category category;
	private String title;
}
2. При формировании данных о снаряжении, которое пользователь взял в прокат, используйте класс RentUnit
public class RentUnit {
	private Equipment[] units;

}
3. Для хранения снаряжения, доступного для проката используйте класс RentStation. Учтите необходимость хранения информации о снаряжении отданном в прокат и находящимся в наличии

4. Реализуйте возможность вывода на консоль отчета о снаряжении, которое было отдано в прокат за определённый интервал времени (за час, сутки, за «сегодня»)
При решении задания необходимо обязательно применить:
перегрузку метода
переопределение метода
абстрактный класс
интерфейс
перечисление
статический метод, статический перегруженный метод
final класс
final метод



1. Информацию о снаряжении, доступном к прокату хранить в текстовом файле.
2. Информацию о снаряжении, отданном в прокат также хранить в текстовом файле.
3. Предусмотреть возможность сохранения отчёта о всех единицах снаряжения, отданных в прокат за определённый период в текстовом файле. В названии файла отразить дату создания отчёта.
Пример: rent_equipment_report_01_04_2017.txt
1. Ivanov Ivan MC1563874 
equipment_info
* equipment_info – подробная информация о снаряжении
4. Предусмотреть возможность поиска информации о конкретном снаряженни согласно параметрам (информация хранится в файле).
Не использовать класс Scanner, сериализацию. Пользоваться только байтовыми, либо символьными потоками.
